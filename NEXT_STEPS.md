# 次にCodexへ頼むと良い作業例

## 優先度A：現場操作の安定化

- iPhone幅での表示崩れ修正
- タップ判定の改善
- 選択中状態の見やすさ改善
- 誤操作防止メッセージの短文化
- 共有テキストの整形

## 優先度B：保存・履歴

- 保存したオーダーの読み込みUI改善
- 過去オーダーの複製
- 試合日・相手別検索
- JSONバックアップ/復元

## 優先度C：本番化

- PWA対応
- ホーム画面追加用アイコン
- GitHub Pagesなどで公開確認
- 必要になってからクラウド保存化

## Codexへの依頼は小分けにする

悪い例：
「全部いい感じにして」

良い例：
「iPhone幅で、回チップと共有ボタンが1画面上部に収まるようにしてください。既存機能は壊さず、CSS中心で修正してください。」
