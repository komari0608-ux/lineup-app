# AGENTS.md

このリポジトリは、少年野球向けの「打順・守備交代プランナー」です。
作業者はこのファイルを必ず読んでから変更してください。

## 最重要方針

- スマホ操作を最優先にする。
- オーダー画面内で操作を完結させる。
- 打順変更、守備変更、イニング変更、選手交代など試合中のリアルタイム操作にはモーダルを使用しない。
- 試合情報、表示設定、打順方式など試合前に整える項目は設定モーダルで扱ってよい。
- ポップアップ、別画面遷移、タップ後に出る大きなメニューは使用しない。
- 守備変更専用カードを復活させない。
- 大きな表形式（全選手×9回）は使用しない。
- 既存の選手登録、保存、共有、打順変更を壊さない。
- 変更は小さく、確認しやすく実装する。

## 現在のアプリ構成

- `index.html` にHTML/CSS/JavaScriptがまとまっている単体HTML試作。
- まずは単体HTMLのまま改善する。
- ユーザーから明示されない限り、React化・Next.js化・Firebase化などの大規模移行はしない。

## 操作仕様

### 初期モード

- 初期オーダー作成時のみ打順変更できる。
- 選手名を2回タップして2人を入れ替える。
- 入れ替え時は選手に紐づく守備情報も一緒に移動する。

### 回選択モード

- 1回〜9回の回チップをタップすると、その回からの守備変更モードになる。
- 回選択中は打順変更不可。
- 守備チップ2タップで守備位置を入れ替える。
- ベンチ選手との交代は可能。
- 打順番号は固定。
- 選手交代・守備変更はその回からの変更として保存する。

## 表示仕様

- 守備履歴は選手名の右側に横並びで表示する。
- 例：`陽向　遊　3回:捕　5回:投`
- 変更がない選手は初期守備のみ表示する。
- 守備位置は数字ではなく、`投・捕・一・二・三・遊・左・中・右・DH・ベ` で表示する。

## 保存・共有

- localStorage を使った端末保存は維持する。
- Web Share API が使える場合は共有メニューを開く。
- 使えない場合はクリップボードコピーにフォールバックする。
- 共有文には守備変更・選手交代を簡潔に反映する。
- 変更がない選手を守備変更欄に出さない。

## 完了条件

変更後は以下を確認すること。

- 初期モードで打順入替できる。
- 回選択中は打順入替できない。
- 回選択中に守備チップ2タップで入替できる。
- ベンチ選手との交代ができる。
- 共有テキストに変更が反映される。
- 保存後に再読み込みしても選手登録・オーダーが保持される。
- iPhone幅の画面でスクロール量が増えすぎていない。
