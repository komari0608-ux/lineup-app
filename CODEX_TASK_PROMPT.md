# Codexへ最初に渡すプロンプト

以下をそのままCodexに貼り付けてください。

---

このリポジトリは、少年野球向けの「打順・守備交代プランナー」です。
まず `README.md`、`AGENTS.md`、`index.html` を読んで、現在の仕様と実装を把握してください。

作業前に、次の3点を短く報告してください。

1. 現在のデータ構造の概要
2. 初期モードと回選択モードの操作フロー
3. 既存仕様を壊さないために注意する箇所

その後、私が依頼する次の改善だけを小さく実装してください。

絶対に守ること：

- モーダルは使わない
- 別画面遷移はしない
- タップメニューは使わない
- 守備変更専用カードを復活させない
- 全選手×9回の大きな表にしない
- スマホでの操作性を最優先にする
- 既存の選手登録、保存、共有、打順変更機能を壊さない
- 9人制について「打順引き継ぎ」という文言は使わない

現在の最新版は v10 です。
大きな作り替えではなく、まずは `index.html` の中で改善してください。
React化、Next.js化、Firebase化、クラウド保存化は、私が明示的に依頼するまで行わないでください。

実装後は、次の確認結果を報告してください。

- 初期モードで打順入替できるか
- 回選択中に打順入替が無効か
- 守備チップ2タップで守備変更できるか
- ベンチ選手との交代ができるか
- 保存・再読み込みでデータが保持されるか
- 共有テキストに守備変更と選手交代が反映されるか
